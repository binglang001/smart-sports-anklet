# 快速开始与部署指南

本文档面向“只看文档就完成部署”的场景编写，覆盖硬件清单、接线方式、GNSS 用户库安装、服务端部署、设备端运行和自启动注意事项。

## 1. 部署目标

完成以下结果即表示部署成功：

- 行空板 M10 能正常启动设备端程序
- Web 控制端可以显示设备在线状态
- 模式切换可同步到网页端
- 运动模式下能够记录步数、GNSS 数据和运动详情

## 3. 接线说明

### 3.1 当前代码默认引脚

当前项目默认引脚配置位于 `client/config.py`：

```python
PIN_DHT11 = "P23"
PIN_TOUCH = "P24"
PIN_KNOB = "P22"
PIN_LED = "P21"
LED_COUNT = 19
```

### 3.2 接线表

| 模块 | 行空板连接 | 说明 |
| --- | --- | --- |
| DHT11 | `DATA -> P23` | 其余接 `VCC` / `GND` |
| 数字触摸传感器 | `SIG -> P24` | 其余接 `VCC` / `GND` |
| 模拟角度传感器 | `SIG -> P22` | 模拟输出接 `P22` |
| WS2812 灯带 | `DIN -> P21` | 其余接电源与地，当前灯珠数为 `19` |
| TEL0157 GNSS | `SDA/SCL -> I2C` | 开关拨到 `I2C`，代码默认地址 `0x20` |
| DFR0760 语音模块 | `SDA/SCL -> I2C` | 与 GNSS 共用 I2C 总线 |

### 3.3 接线注意事项

- 所有模块必须**共地**
- GNSS 和语音模块均为 I2C 设备，可以并联到同一组 `SDA` / `SCL`
- GNSS 模块测试时需连接天线，并保证天线陶瓷面朝上
- 若 WS2812 灯带较长或亮度较高，建议单独供电，但必须与行空板共地

## 4. 服务端部署

### 4.1 安装依赖

在项目根目录执行：

```bash
pip install -r requirements.txt
```

### 4.2 启动 Flask 服务端

```bash
python server/server.py
```

也可以使用启动脚本：

- Windows：`start_server.bat`
- Linux / macOS：`start_server.sh`

### 4.3 访问地址

- 控制台：`http://localhost:5000/`
- 历史记录：`http://localhost:5000/history`

如果部署在云服务器，请确保开放 `5000` 端口，并将设备端 `SERVER_URL` 改为对应公网地址。

## 5. 设备端软件准备

### 5.1 推荐环境

推荐直接使用行空板官方 Python 运行环境或 Mind+ Python 模式环境。项目设备端依赖以下运行库：

- `pinpong`
- `unihiker`
- `requests`
- `pinpong.libs.dfrobot_speech_synthesis`
- GNSS 用户库（通过 Mind+ 安装）

### 5.2 GNSS 模块驱动库安装方式

TEL0157 的 Python 驱动建议通过 Mind+ 安装，步骤如下：

1. 打开 **Mind+**，切换到 **Python 模式**
2. 添加 **行空板扩展**
3. 在用户库中搜索 **GNSS 模块**
4. 添加对应的 GNSS 用户库
5. 连接行空板
6. 在图形化界面中使用一次“初始化 GNSS”相关积木块并运行

完成后，Mind+ 会把相关 GNSS 库安装到行空板环境中。当前项目中的 `client/services/gnss_manager.py` 会自动搜索这些常见安装路径并尝试加载驱动。

### 5.3 GNSS 模块使用注意事项

- TEL0157 开关必须拨到 **I2C** 一侧
- 首次搜星可能需要几十秒，请在**户外开阔场地**测试，模块绿灯常亮即为搜星完成
- 天线白色陶瓷面朝天空
- 室内通常无法获得稳定卫星信号

## 6. 设备端配置

打开 `client/config.py`，至少确认以下参数：

```python
SERVER_URL = "http://你的服务端IP:5000"

PIN_DHT11 = "P23"
PIN_TOUCH = "P24"
PIN_KNOB = "P22"
PIN_LED = "P21"
LED_COUNT = 19
```

如果你的接线与默认配置不同，请同步修改这些参数。

## 7. 部署并运行设备端

### 7.1 拷贝代码

将仓库中的 `client/` 目录部署到行空板运行环境中。建议目录保持完整，至少包含：

- `client/main.py`
- `client/config.py`
- `client/sensors/`
- `client/services/`
- `client/ui/`
- `client/utils/`

### 7.2 手动运行

在设备端执行：

```bash
cd client
python main.py
```

程序启动后会尝试：

- 初始化 ICM20689 传感器
- 初始化 DHT11、触摸输入、旋钮和 WS2812 灯带
- 初始化 I2C 语音模块
- 初始化 GNSS 管理器
- 与服务端同步状态

## 8. 自启动注意事项

项目当前通过字符设备文件访问加速度传感器，`client/sensors/icm20689.py` 依赖的设备路径为：

```text
/dev/icm20689
```

因此如果你设置开机自启动，**不能在系统刚上电时立即直接启动 `client/main.py`**，而是要先等待 `/dev/icm20689` 出现。

### 8.1 推荐的等待方式

可以使用一个简单的启动包装脚本：

```bash
#!/bin/sh
while [ ! -e /dev/icm20689 ]; do
  sleep 1
done

cd /root/smart-sports-belt/client
python main.py
```

如果使用 systemd，也建议把这段等待逻辑放进 `ExecStartPre` 或包装脚本中。

## 9. 验收步骤

部署完成后，按下面顺序验证：

1. 服务端启动正常，无报错
2. 浏览器能打开 `http://服务器IP:5000/`
3. 设备端能正常启动并显示 UI
4. 网页端能看到设备在线状态
5. 切换模式后网页端状态同步更新
6. 户外进入运动模式后，GNSS 能完成搜星并出现运动记录

## 10. 常见问题

### 10.1 设备端启动时报硬件相关错误

- 检查是否在真正的行空板环境运行
- 检查相关运行库是否已安装
- 检查 `/dev/icm20689` 是否存在

### 10.2 GNSS 无法读取数据

- 检查 TEL0157 是否拨到 `I2C` 模式
- 检查 Mind+ 是否已安装 GNSS 用户库
- 检查天线是否连接，是否在户外开阔环境，模块是否绿灯常亮
- 检查 `SDA` / `SCL` 接线是否正确

### 10.3 网页端没有设备数据

- 检查 `SERVER_URL` 是否正确
- 检查服务端 IP 是否可被设备访问
- 检查服务端是否正常运行在 `5000` 端口

### 10.4 灯带或输入模块无反应

- 检查默认引脚是否与实际接线一致
- 检查所有模块是否共地
- 检查供电是否稳定
