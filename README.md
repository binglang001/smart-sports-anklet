## 项目简介

智碳腰带是一套面向校园运动、日常健康管理和安全提醒场景的可穿戴原型系统。项目由三部分组成：

- **设备端**：运行在行空板 M10 上，负责传感器采集、本地显示、语音播报和模式切换
- **服务端**：基于 Flask 提供 REST API、状态存储和历史记录管理
- **Web 控制端**：用于查看设备状态、历史数据、运动记录与远程控制

当前版本主要包含三种运行模式：

- **生活模式**：显示环境信息、姿态状态与基础统计
- **运动模式**：记录步数、距离、GNSS 轨迹、速度与配速
- **会议模式**：降低干扰，适合上课、会议等场景

## 功能特性

- **50Hz 高频采样**：基于行空板 M10 板载 ICM20689 进行运动数据采集
- **步数检测**：针对腰部佩戴场景进行步数统计
- **姿态识别**：识别站立、坐姿、躺卧、移动等状态
- **跌倒检测**：异常姿态与加速度变化触发提醒逻辑
- **GNSS 运动记录**：支持速度、配速、轨迹点和 UTC 时间获取
- **温湿度监测**：采集环境温湿度数据
- **语音播报**：模式切换、异常提醒与状态提示
- **LED 灯效提示**：使用 WS2812 灯带进行状态显示与告警
- **网页控制**：支持控制端查看设备状态、历史记录和运动详情
- **离线缓存**：网络异常时本地缓存，恢复连接后自动同步

## 硬件需求

### 材料清单

| 名称 | 规格 | 数量 | 主要用途 |
| --- | --- | --- | --- |
| 行空板 M10 | 主控 | 1 块 | 数据采集、分析计算、UI 显示、网络连接 |
| GNSS 模块 | TEL0157 | 1 个 | 配速、轨迹、时间获取 |
| 温湿度传感器 | DHT11 | 1 个 | 环境数据监测 |
| 语音合成模块 | DFR0760 | 1 个 | 语音提示、状态播报 |
| RGB 灯带 | WS2812 | 1 组 | 状态指示、紧急警报 |
| 数字触摸传感器 | 电容触摸 | 1 个 | 设备交互 |
| 旋钮 | 模拟角度传感器 | 1 个 | 灯带控制 |
| 腰包 | 任意运动腰包 | 1 个 | 穿戴安装 |
| 本地或云端服务器 | 云服务器或电脑设备 | 1 台 | 远程控制、历史记录与数据展示 |

### 当前代码默认接线

以下接线对应当前 `client/config.py` 中的默认配置：

| 模块 | 接口类型 | 行空板连接 | 说明 |
| --- | --- | --- | --- |
| DHT11 | 单总线数字输入 | `P23` | 数据脚接 `P23`，其余接 `VCC` / `GND` |
| 数字触摸传感器 | 数字输入 | `P24` | 触摸输出接 `P24` |
| 模拟角度传感器 | 模拟输入 | `P22` | 旋钮信号脚接 `P22` |
| WS2812 灯带 | 单线数据输出 | `P21` | 数据输入接 `P21`，当前灯珠数量为 `19` |
| GNSS TEL0157 | I2C | `SDA` / `SCL` | 拨到 `I2C` 模式，代码默认地址 `0x20` |
| DFR0760 语音模块 | I2C | `SDA` / `SCL` | 与 GNSS 共用 I2C 总线 |

部署时请额外注意：

- 所有模块必须**共地**
- GNSS 与语音模块均为 I2C 设备，可并联在同一组 `SDA` / `SCL` 上
- GNSS 模块需要连接外置天线，并在户外开阔环境测试
- 若 WS2812 灯带供电电流较大，建议单独供电，但必须与主控共地

更详细的接线与部署流程请直接查看 `QUICKSTART.md`。

## 快速开始

### 1. 启动服务端

```bash
pip install -r requirements.txt
python server/server.py
```

启动脚本：

- Windows：`start_server.bat`
- Linux / macOS：`start_server.sh`

默认访问地址：

- 控制台：`http://localhost:5000/`
- 历史记录：`http://localhost:5000/history`

### 2. 配置设备端地址

修改 `client/config.py`：

```python
SERVER_URL = "http://你的服务端IP:5000"
```

例如：

```python
SERVER_URL = "http://192.168.1.100:5000"
```

### 3. 运行设备端

将 `client/` 目录部署到行空板环境，在设备端执行：

```bash
cd client
python main.py
```

## 部署教程

如果你希望仅看文档就完成部署，建议按以下顺序执行：

1. 按照 `QUICKSTART.md` 准备全部硬件并完成接线
2. 在 Mind+ 的 Python 模式中安装 GNSS 用户库
3. 在电脑或云服务器上启动 Flask 服务端
4. 修改 `client/config.py` 中的 `SERVER_URL` 和必要参数
5. 将 `client/` 部署到行空板并运行 `main.py`
6. 打开 Web 页面，验证设备状态、模式切换与数据回传
7. 如果需要自启动，务必先等待 `/dev/icm20689` 字符设备出现后再启动客户端

## 使用说明

### 设备基本使用

- 开机后进入默认运行状态
- 通过触摸输入、旋钮或网页端切换模式
- 运动前切换到**运动模式**，等待 GNSS 搜星完成后再开始测试
- 运动结束后在网页端查看历史记录和运动详情

### 模式说明

| 模式 | 主要功能 |
| --- | --- |
| 生活模式 | 环境显示、姿态状态、基础数据统计 |
| 运动模式 | 步数、距离、GNSS 速度、配速、轨迹记录 |
| 会议模式 | 降低显示与播报干扰 |

## 故障排查

### 设备端启动失败

- 检查行空板运行环境是否完整
- 检查 `pinpong`、`unihiker`、`requests` 等设备端依赖是否可用
- 检查 `/dev/icm20689` 是否已经出现

### GNSS 无法工作

- 确认 TEL0157 已拨到 `I2C` 模式
- 确认天线已接好且模块处于户外开阔环境，模块已搜星完成并绿灯常亮
- 确认已在 Mind+ 中安装并初始化过 GNSS 用户库
- 确认 I2C 总线接线正确，且未出现地址冲突

### 网页端没有数据

- 检查 `SERVER_URL` 是否正确
- 检查设备和服务端是否处于可互通网络
- 检查服务端进程是否正常运行，端口 `5000` 是否可访问
